# 小范围交付版发布验收

本文档定义 `自媒体视频工作台` 第一版小范围交付前必须通过的本地、桌面端、真实 API、导出一致性和恢复测试。测试报告可以记录脱敏服务状态、模型名、错误原因和文件路径，但不能记录 API Key 原文。

## 发布命令

```bash
npm run release:check
npm run release:ui
npm run release:e2e:real
```

- `release:check`：执行 TypeScript 类型检查、ESLint、Vitest、生产构建和 Prettier 检查。
- `release:ui`：启动 Electron 桌面端并用 CDP 自动验收新建、重命名、删除、设置弹窗、服务标签、成品预览比例和封面预览比例。
- `release:e2e:real`：使用真实服务配置创建六种视频模式任务，并为每个任务生成竖屏和横屏输出。该命令会消耗真实 API 和 HeyGen 额度。

单模式补测可以设置 `RELEASE_E2E_MODES`，多个模式用英文逗号分隔，例如：

```powershell
$env:RELEASE_E2E_MODES = "mixed-cut"
npm run release:e2e:real
Remove-Item Env:RELEASE_E2E_MODES
```

## 必须真实跑通的六种模式

1. 预设数字人口播。
2. 商品/带货视频。
3. 图片口型同步。
4. 个人 IP 视频。
5. 爆款视频复刻。
6. 混剪视频。

每个模式必须生成：

- 真实 MP4 成片。
- 封面文件。
- 字幕文件。
- 发布资料包 manifest。
- 竖屏 `portrait-9-16` 和横屏 `landscape-16-9` 两个输出变体。

## 发布阻塞项

- HeyGen API 必须能自动读取可用 Avatar。设置页不要求手填 Avatar ID，任务内应能选择 Avatar。
- HeyGen Voice ID 必须有可用兜底或任务内可选项，不能再因为 `voice_id is required` 阻断生成。
- 成片 MP4 必须内嵌预览中设置的字幕、画面标题、字体、颜色和百分比位置。
- 如果 HeyGen 没有返回可用字幕，ASR 必须返回带时间轴的字幕；纯文本转写不能作为正式字幕。
- 混剪视频必须使用上传素材生成可导出的混剪成品，不能用数字人口播占位。
- 设置页保存后必须显示真实测试结果；失败时显示服务返回的真实失败原因。
- 桌面端不能白屏、闪退、遮挡关键按钮，预览比例必须正确。

## API 与模型要求

| 功能环节                | 服务配置                      | 发布前要求                                            |
| ----------------------- | ----------------------------- | ----------------------------------------------------- |
| 文案分析/改写/原创生成  | LLM，OpenAI 兼容              | 能列出或保存模型，能完成脚本生成                      |
| 商品人物图/换装图       | 图片生成，OpenAI 兼容         | 能完成商品图导入后的 presenter image 生成             |
| 预设数字人/图片口型同步 | HeyGen                        | 能读取 Avatar，能生成真实 lip-sync 视频               |
| 字幕兜底                | ASR 或可复用 LLM 音频转写接口 | 必须通过真实 `/audio/transcriptions` 探针并返回时间轴 |
| 混剪基础视频            | 本地 FFmpeg                   | 能从上传视频或图片素材生成竖屏/横屏基础视频           |
| 最终导出                | 本地 FFmpeg                   | 能将字幕和画面标题烧录进 MP4                          |

## 数据、日志和备份

- 默认数据目录：`%APPDATA%\自媒体视频工作台`。
- 开发时可用 `DHS_APP_DATA_DIR` 指定隔离数据目录。
- 真实验收脚本默认从 `%APPDATA%\自媒体视频工作台` 复制服务配置到 `tmp/release-e2e-real-*` 的隔离目录；也可以用环境变量提供 API 配置。
- 备份时应包含项目目录、Git 仓库、`%APPDATA%\自媒体视频工作台` 数据目录，以及 Codex 对话备份目录；不要把 API Key 写入仓库或 Markdown 文档。
- 恢复时先安装 Node 依赖，再重新填入 API Key 或恢复本机凭据文件，然后执行 `npm run release:check` 和一条短样片真实生成。

## 小范围交付包要求

小范围交付可以先使用可复制运行包；正式给用户前必须补齐：

- Windows 可运行包或安装包。
- 默认数据目录说明。
- API 配置引导。
- 错误日志位置说明。
- 备份/恢复说明。
- 一条干净目录下重新配置 API 并生成视频的验收记录。
