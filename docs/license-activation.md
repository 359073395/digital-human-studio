# 离线激活码授权说明

跑量自媒体视频工作台第一版采用离线激活码授权，不依赖服务器。软件内只内置公钥；私钥只保存在开发者本机，用来给内部试用电脑签发激活码。

## 首次生成密钥

```bash
npm run license:keygen
```

生成结果：

- 私钥：`data/license/private-key.pem`
- 公钥源码：`src/main/license/licensePublicKey.ts`

`data/` 已在 `.gitignore` 中，私钥不会提交到 GitHub。请把私钥单独备份到安全位置；如果丢失，只能重新生成密钥，旧激活码会全部失效。

## 打开授权码生成器

```bash
npm run license:tool
```

生成器是一个本地小窗口。输入用户激活页显示的机器码、授权对象和有效天数，即可生成 `PLV1...` 激活码，并可一键复制。

## 命令行签发

```bash
npm run license:create -- --machine ABCD-1234-EFGH-5678 --holder 张三 --days 90
```

常用有效期：

- 30 天：内部短测。
- 90 天：小范围试用。
- 365 天：长期内部使用。

## 用户激活流程

1. 用户打开软件。
2. 未激活时只显示“激活软件”页面。
3. 用户复制机器码发给你。
4. 你用授权码生成器或命令行生成激活码。
5. 用户粘贴激活码并点击激活。

激活后才能进入主工作台、设置 API、创建任务、生成视频和导出成品。

## 开发测试绕过

自动化测试可使用：

```bash
DHS_LICENSE_TEST_BYPASS=1 npm run dev
```

绕过只在未打包开发运行环境生效，正式安装包不会因为这个环境变量跳过激活。

## GitHub 规则

- 可以提交：公钥源码、授权校验代码、授权说明文档、生成器脚本。
- 禁止提交：`data/license/private-key.pem`、真实激活码、客户机器码名单、授权对象隐私信息。
