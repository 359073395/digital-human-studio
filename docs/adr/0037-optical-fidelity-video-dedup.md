# ADR 0037: 光学保真视频去重处理

## Status

Accepted

## Context

视频去重处理在 `v1.0.12` 已经简化了界面，只保留导入视频、处理强度、输出比例和一键输出。这个方向适合内部试用和新手操作，但旧实现容易被理解成普通裁边、调色、改编码或评分报告。

用户明确要求：之前学习到的去重知识必须真正用于软件，不能只停留在聊天记录里。去重应包含光学偏移、动态裁切、镜头畸变、透视扰动、帧重采样、像素扰动、音频轻扰动，以及后续可选的视频模型片段重构。

## Decision

保持用户界面简单，只提供三档处理强度：

- 轻度：轻微光学偏移，观感最接近。
- 标准：动态裁切 + 光学扰动，默认推荐。
- 强处理：透视/镜头扰动更明显。

内部处理链路按强度组合 FFmpeg 滤镜：

- time-based dynamic crop/pan，用于局部动态裁切和光学偏移。
- subtle rotate，用于模拟轻微拍摄角度变化。
- lenscorrection，用于轻微镜头畸变。
- perspective，用于标准和强处理的轻微透视扰动。
- fps/GOP/CRF/metadata changes，用于帧重采样和编码结构变化。
- noise/unsharp/tblend，用于像素层变化和强处理下的轻微帧混合。
- highpass/lowpass/volume/aecho，用于不明显改变听感的音频轻扰动。

内置知识库和生产工作流提示必须同步这套规则。去重模式不再把评分报告作为用户必须理解或操作的步骤，评分相关结构只保留为内部兼容和调试能力。

## Consequences

- 新手仍然只需要选择轻度、标准或强处理。
- 处理后视频应尽量保持原视频观感，不使用明显镜像、粗暴边框或一眼可见的夸张变形。
- 混剪仍只负责素材组合；需要二次处理时，把混剪成片导入视频去重处理模式。
- 如果未来接入 Seedance、即梦、可灵或其它 V2V 模型，高风险片段重构应接在这套保真去重流程之后，而不是替换当前本地处理链路。
